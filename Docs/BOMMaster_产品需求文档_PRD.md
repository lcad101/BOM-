# BOM管理系统 - 产品需求文档 (PRD)

> 版本: v1.0 | 日期: 2026-05-28

---

## 1. 项目概述

### 1.1 项目名称

BOMMaster（暂定）

### 1.2 目标用户

| 角色 | 职责描述 |
|------|---------|
| 硬件工程师 | 创建和维护BOM结构，管理元器件选型和替代料 |
| 采购人员 | 查看BOM物料清单，跟踪替代料信息，进行采购决策 |
| NPI项目经理 | 管理项目进度，审核BOM变更，对比版本差异 |
| 系统管理员 | 管理用户权限、系统配置和数据备份 |

### 1.3 核心痛点

1. **版本混乱**：传统Excel管理BOM时，多人复制修改导致版本分支混乱，无法追溯哪个是最新版本，历史修改记录完全丢失。
2. **替代料追踪困难**：元器件缺货时需要替代料，但Excel中替代料关系散落在不同文件和邮件中，无法快速查询某颗料有哪些替代方案、替代料的验证状态如何。
3. **多人协作冲突**：多人同时编辑同一份BOM的Excel文件，经常出现覆盖、丢失数据的情况，缺乏并发控制和变更通知机制。
4. **层级关系不直观**：PCBA到单板到元器件的多层级装配关系在Excel中只能用缩进表示，无法直观展示树状结构，容易遗漏或重复。

### 1.4 平台要求

- Windows 10/11 桌面应用，支持离线使用
- 本地SQLite数据库，无需服务器部署
- 通过Tauri打包为轻量级桌面应用（安装包<20MB）

## 2. 用户故事与验收标准

### US-001: 创建多层级BOM

- **用户故事**：作为一个硬件工程师，我想要创建多层级BOM，以便于反映PCBA、单板、元器件的真实父子装配关系。
- **优先级**：P0 (Must have)
- **前置条件**：用户已创建一个项目。
- **验收标准 (AC)**：
  1. 提供可视化树状图展示BOM层级，支持展开/折叠操作。
  2. 支持在任意节点右键添加"子节点"（下级BOM）或"挂料"（叶子节点）。
  3. BOM层级最多支持10级深度，超过时需弹窗警告。
  4. 同一父节点下，不允许添加重复的子BOM或元器件型号。
  5. 支持拖拽调整节点层级关系。
  6. 每个节点显示：型号、数量、单位、备注等基本信息。
- **异常流**：若用户尝试添加超过10级的节点，拦截操作并提示"已达最大层级"。

### US-002: 导入Excel BOM

- **用户故事**：作为一个硬件工程师，我想要通过上传Excel文件导入BOM，以便于快速将旧系统数据迁移至本系统。
- **优先级**：P0 (Must have)
- **前置条件**：用户已创建目标项目和BOM版本。
- **验收标准 (AC)**：
  1. 支持拖拽或点击上传 `.xlsx` / `.xls` 文件。
  2. 上传后进入"字段映射"页面，用户需将Excel列名（如"Part Number"）映射为系统字段（如"型号"）。
  3. 提供数据预览表格，校验必填项（型号、数量），缺失必填项的行标红提示。
  4. 用户确认映射无误后，点击"确认导入"写入数据库。
  5. 导入完成后显示导入结果摘要（成功行数、失败行数、跳过行数）。
  6. 支持导入模板下载，模板包含所有系统字段的列名。
- **异常流**：文件格式不支持时提示"仅支持.xlsx和.xls格式"；必填字段未映射时禁用"确认导入"按钮。

### US-003: BOM版本管理

- **用户故事**：作为一个硬件工程师，我想要对BOM进行版本管理，以便于追踪每次修改的历史记录，并能在需要时回退到之前的版本。
- **优先级**：P0 (Must have)
- **前置条件**：用户已创建项目并保存了至少一个BOM版本。
- **验收标准 (AC)**：
  1. 每次保存BOM时自动生成新版本，版本号格式为 `v1.0`、`v1.1`、`v2.0`。
  2. 版本列表显示：版本号、创建时间、创建人、变更摘要。
  3. 支持查看任意历史版本的BOM完整内容（只读模式）。
  4. 支持"基于历史版本创建新BOM"操作，即从某个旧版本派生新版本。
  5. 版本状态分为：草稿(Draft)、已发布(Released)、已归档(Archived)。
  6. 已发布版本不可直接修改，需创建新版本进行变更。
- **异常流**：尝试修改已发布版本时，提示"该版本已发布，请创建新版本进行修改"。

### US-004: 替代料管理

- **用户故事**：作为一个硬件工程师，我想要为BOM中的元器件添加替代料，以便于在原型号缺货时快速找到替代方案。
- **优先级**：P0 (Must have)
- **前置条件**：BOM中已存在至少一个元器件节点。
- **验收标准 (AC)**：
  1. 在元器件节点上右键选择"添加替代料"，弹出替代料编辑对话框。
  2. 替代料信息包含：替代型号、厂商、验证状态（未验证/验证中/已验证/不推荐）、备注。
  3. 一个元器件可添加多个替代料，按优先级排序（1=首选替代）。
  4. 在BOM树状图中，有替代料的元器件节点显示特殊图标标识。
  5. 点击元器件节点时，侧边面板展示该元器件的所有替代料列表。
  6. 支持将替代料"提升为正式料"操作，即替换原元器件。
- **异常流**：替代型号与原型号相同时，提示"替代料不能与原件型号相同"。

### US-005: BOM导出

- **用户故事**：作为一个采购人员，我想要将BOM导出为Excel文件，以便于发送给供应商进行报价。
- **优先级**：P1 (Should have)
- **前置条件**：当前BOM版本已保存。
- **验收标准 (AC)**：
  1. 支持导出为 `.xlsx` 格式。
  2. 导出时可选择导出范围：仅顶层、展开所有层级、仅叶子节点。
  3. 导出时可选择包含字段：型号、数量、单位、厂商、描述、替代料信息。
  4. 多层级BOM导出时，使用缩进表示层级关系。
  5. 导出文件名格式：`{项目名}_{BOM名}_v{版本号}_{日期}.xlsx`。
- **异常流**：BOM为空时导出按钮置灰，提示"当前BOM无数据可导出"。

### US-006: 高级搜索与筛选

- **用户故事**：作为一个硬件工程师，我想要在BOM中快速搜索元器件，以便于定位特定型号或查看某类元器件的使用情况。
- **优先级**：P1 (Should have)
- **前置条件**：BOM中已有数据。
- **验收标准 (AC)**：
  1. 提供全局搜索框，支持按型号、描述、厂商进行模糊搜索。
  2. 搜索结果高亮显示匹配的节点，并自动展开其父节点。
  3. 支持按元器件类别、厂商、是否有替代料等条件进行筛选。
  4. 搜索结果面板显示所有匹配项列表，点击可跳转到BOM树中对应节点。
  5. 支持跨项目搜索元器件使用情况（哪些BOM使用了该型号）。
- **异常流**：无匹配结果时显示"未找到匹配的元器件"。

### US-007: 项目管理

- **用户故事**：作为一个NPI项目经理，我想要创建和管理多个项目，以便于将不同产品的BOM进行分类管理。
- **优先级**：P0 (Must have)
- **前置条件**：用户已登录系统。
- **验收标准 (AC)**：
  1. 支持创建项目，项目信息包含：项目名称、项目编号、描述、负责人、创建时间。
  2. 项目列表以卡片形式展示，显示项目名称、BOM数量、最近修改时间。
  3. 支持项目搜索和排序（按名称、创建时间、修改时间）。
  4. 支持项目归档操作，归档后项目不在活跃列表中显示，但可查看。
  5. 每个项目下可创建多个BOM。
- **异常流**：项目名称重复时提示"项目名称已存在"。

### US-008: BOM对比

- **用户故事**：作为一个NPI项目经理，我想要对比两个BOM版本的差异，以便于审核变更内容。
- **优先级**：P1 (Should have)
- **前置条件**：同一BOM至少存在两个版本。
- **验收标准 (AC)**：
  1. 支持选择同一BOM的任意两个版本进行对比。
  2. 对比结果以并排视图展示，差异项高亮标记。
  3. 差异类型分为：新增节点（绿色）、删除节点（红色）、修改节点（黄色）。
  4. 修改节点显示具体变更字段（如数量从5改为10）。
  5. 支持导出对比报告为Excel文件。
- **异常流**：选择同一版本对比时提示"请选择不同的版本进行对比"。

### US-009: 变更历史

- **用户故事**：作为一个硬件工程师，我想要查看BOM的完整变更历史，以便于追溯每次修改的详细信息。
- **优先级**：P1 (Should have)
- **前置条件**：BOM已有变更记录。
- **验收标准 (AC)**：
  1. 变更历史列表按时间倒序排列，显示：变更时间、操作人、变更类型、变更摘要。
  2. 变更类型包含：新增节点、删除节点、修改属性、添加替代料、移除替代料。
  3. 点击某条变更记录，展开显示变更前后的详细对比。
  4. 支持按变更类型、时间范围、操作人进行筛选。
- **异常流**：无变更记录时显示"暂无变更历史"。

### US-010: 数据导入校验与预览

- **用户故事**：作为一个硬件工程师，我想要在导入数据前进行校验和预览，以便于确保导入数据的准确性。
- **优先级**：P2 (Nice to have)
- **前置条件**：用户已上传Excel文件并完成字段映射。
- **验收标准 (AC)**：
  1. 校验规则包含：必填字段检查、数据类型校验、重复数据检测、层级关系验证。
  2. 校验结果分为：错误（必须修复）、警告（建议修复）、通过。
  3. 预览表格中错误行标红、警告行标黄、通过行标绿。
  4. 支持忽略警告继续导入，但错误必须修复。
  5. 提供批量修复功能：对空值填充默认值、对重复项自动去重。
- **异常流**：存在未修复的错误时，"确认导入"按钮禁用。

## 3. 非功能性需求

### 3.1 性能要求

| 场景 | 指标 | 目标值 |
|------|------|--------|
| BOM树加载 | 包含1000个节点的BOM树渲染时间 | ≤1秒 |
| Excel导入 | 包含10,000行的BOM数据解析与渲染 | ≤3秒 |
| 搜索响应 | 全局搜索返回结果时间 | ≤500ms |
| 版本对比 | 两个版本对比计算时间 | ≤2秒 |
| 导出Excel | 导出包含5000行的BOM | ≤2秒 |
| 应用启动 | 冷启动到可交互时间 | ≤3秒 |

### 3.2 数据安全

1. 本地数据需自动备份，每日增量备份、每周全量备份。
2. 备份文件存储在应用数据目录的 `backups/` 子目录下。
3. 支持手动触发备份和从备份恢复。
4. 删除操作需二次确认，项目删除支持软删除（30天内可恢复）。
5. 数据库文件采用SQLite加密扩展（SQLCipher）进行加密存储。

### 3.3 可用性

1. 操作界面支持中文和英文两种语言。
2. 提供操作引导和工具提示（Tooltip）。
3. 常用操作支持键盘快捷键（Ctrl+S保存、Ctrl+Z撤销、Ctrl+F搜索）。
4. 支持撤销/重做操作（Undo/Redo），操作历史栈深度为50步。

### 3.4 兼容性

1. 操作系统：Windows 10 (1809+)、Windows 11。
2. 屏幕分辨率：最低1366x768，推荐1920x1080。
3. 不依赖网络连接，完全支持离线使用。

## 4. UI/UX设计要求

### 4.1 整体布局

- 采用经典的三栏布局：左侧项目导航树、中间BOM编辑区、右侧属性面板。
- 顶部工具栏包含：项目操作、BOM操作、导入导出、搜索等常用功能。
- 底部状态栏显示：当前项目、BOM版本、节点数量、保存状态。

### 4.2 交互规范

1. 所有破坏性操作（删除、覆盖导入）必须弹出确认对话框。
2. 长时间操作（导入、导出、对比）显示进度条，支持取消。
3. 表格支持列宽拖拽调整、列排序、列筛选。
4. 树状图支持键盘导航（上下箭头移动、左右箭头展开折叠、Enter编辑）。
5. 右键上下文菜单根据节点类型动态显示可用操作。

### 4.3 视觉规范

- 主色调：专业蓝 (#409EFF)，辅助色：成功绿 (#67C23A)、警告橙 (#E6A23C)、危险红 (#F56C6C)。
- 字体：中文使用微软雅黑，英文使用Segoe UI，代码使用Consolas。
- 间距基准单位：8px，所有间距为8的倍数。

## 5. 数据迁移与部署

### 5.1 数据迁移

1. 提供Excel模板下载功能，模板包含所有系统字段的列名和示例数据。
2. 导入时自动检测并处理重复数据（按型号+厂商去重）。
3. 支持分批导入，单次导入上限为50,000行。

### 5.2 部署要求

1. 安装包格式：Windows MSI安装包 + 便携版（免安装）。
2. 安装包体积控制在20MB以内。
3. 首次安装后自动创建桌面快捷方式。
4. 支持自动更新检测（需联网），更新包采用增量更新。

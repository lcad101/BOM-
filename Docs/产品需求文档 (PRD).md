# BOM管理系统 - 产品需求文档 (PRD)

## 1. 项目概述
- **项目名称：** BOMMaster (暂定)
- **目标用户：** 硬件工程师、采购人员、NPI项目经理
- **核心痛点：** 解决传统Excel管理BOM导致的版本混乱、替代料追踪困难、多人协作冲突问题。
- **平台要求：** Windows 10/11 桌面应用，支持离线使用。

## 2. 用户故事与验收标准 (核心功能)

### US-001: 创建多层级BOM
- **用户故事：** 作为一个硬件工程师，我想要创建多层级BOM，以便于反映PCBA、单板、元器件的真实父子装配关系。
- **优先级：** P0 (Must have)
- **前置条件：** 用户已创建一个项目。
- **验收标准 (AC)：**
  1. 提供可视化树状图展示BOM层级。
  2. 支持在任意节点右键添加"子节点"（下级BOM）或"挂料"（叶子节点）。
  3. BOM层级最多支持 10 级深度，超过时需弹窗警告。
  4. 同一父节点下，不允许添加重复的子BOM或元器件型号。
- **异常流：** 若用户尝试添加超过10级的节点，拦截操作并提示"已达最大层级"。

### US-002: 导入Excel BOM
- **用户故事：** 作为一个硬件工程师，我想要通过上传Excel文件导入BOM，以便于快速将旧系统数据迁移至本系统。
- **优先级：** P0 (Must have)
- **验收标准 (AC)：**
  1. 支持拖拽或点击上传 `.xlsx` / `.xls` 文件。
  2. 上传后进入"字段映射"页面，用户需将Excel列名（如"Part Number"）映射为系统字段（如"型号"）。
  3. 提供数据预览表格，校验必填项（型号、数量），缺失必填项的行标红提示。
  4. 用户确认映射无误后，点击"确认导入"写入数据库。

## 3. 非功能性需求
- **性能：** 导入包含 10,000 行的BOM数据，解析与渲染时间不超过 3 秒。
- **数据安全：** 本地数据需自动备份，防止误删。
